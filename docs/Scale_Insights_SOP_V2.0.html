5. Implementation Methodology

Data Range Analysis

We aim to analyze data across different date ranges to read both the latest trends and the broader performance picture.

Our approach prioritizes shorter date ranges to make more immediate, data-driven decisions. If the data is not enough in shorter date range, then we analyze a longer period.

From 7 days, 14 days, 30 days, to 60 days.

Days to Exclude

Exclude:

  • The past 2 days due to Amazon Ads Attribution
  • Holidays like Prime Day, BFCM and the 2 days before and after each holiday

We don't want to have outlier dates in our data set since Scale Insights could increase/decrease the bids after analyzing these dates.

Days to Wait/Observe After Making a Change

If we take action every day, we'll be analyzing nearly the same data with only a one-day difference, which won't lead to meaningful changes.

Our approach is to take action, observe the results, and then act again based on the new data.

In our rules, we set a 7-day waiting period after making a bid change for each target.

  • If a keyword meets the set criteria, Scale Insights will adjust the bid and wait 7 days before re-evaluating that keyword
  • If the criteria aren't met, Scale Insights will continue analyzing the keyword daily until the conditions are satisfied

Rule Stacking and Priority

Stacking: We can "stack" rules on top of each other. This means that we can connect to one ad group different rules so Scale Insights will analyze multiple rules in order of priority.

Priority: Once we have multiple rules, Scale Insights needs to understand which rule they need to be analyzed first. The rule with the highest priority will be the first one to be analyzed, if the criteria are not met, then Scale Insights will analyze the rule with the second highest priority and so on.

8. Rule Types & Setup

Bidding Rules

Goal: Bid optimization is a crucial element in effectively optimizing and scaling our ad account. We will leverage Scale Insights' automated rules to achieve this.

  • Flexibility in customization
  • Full control over bid adjustments
  • Transparency in rule execution
  • Precision required in setup

Negative Rules

The end-goal is automating the process of adding negative rules only to discovery campaigns and auto campaigns.

We only add Negative Targeting rules to Discovery and Auto campaigns since the idea behind these campaigns is to capture new traffic, and therefore some of the Search Terms would be irrelevant to our business.

Key Setup Summary:

  • How many clicks? Based on Conversion Rate and Tolerance of Error
  • Data range to analyze? 7, 14, 30 and 60 days - in this order
  • Days to Exclude? Last 2 and holidays (+ 2 days before and after)
  • Days to Observe? 7 Days

9. Copy-Paste Ready Filters

General Exclusions:

Portfolio name does NOT contain: DEF, BRAND, BKWS, PROTECT Campaign name does NOT contain: DEF, BRAND, BKWS, PROTECT Status = Enabled

Negative Rules Additional:

Campaign name does NOT contain: EX, SKC, -EX Ad group name does NOT contain: EXEC, SKC

Why These Exclusions?

  • General Exclusions: Protect brand/defense campaigns from non-brand optimization logic
  • Negative Rules Additional: Prevent adding negatives to winner campaigns (EX/SKC) that are already profitable
  • Status = Enabled: Only applies rules to active campaigns/ad groups

11. Defense Playbook

BKWS (Branded Keywords Search)

Campaigns bidding on your own brand terms + close variants/misspellings.

Goal: Block competitors on your name and keep message/TOS placement
Approach: Use CPC ceilings; maintain modest TOS multiplier; focus on coverage, not expansion
Negatives: Only block obvious junk intent (customer service, refund, manual)

Def PDP (Defensive Product Detail Page)

SP campaigns targeting your own ASINs so competitor ads don't steal shoppers.

Goal: Cross-sell/upsell within your brand; prevent competitor takeover
Approach: Prioritize hero ASINs; keep budgets modest but always-on
Negatives: Add negative ASINs/categories for serial bleeders with 0 orders

Why Exclude Defense from Normal Rules?

  • Different Intent: Branded traffic converts high and is not "new" demand
  • Avoid Overspend: Low ACOS on brand can trick automation into over-bidding
  • Protect Coverage: Negative/harvesting rules can accidentally block brand terms
  • Cleaner Measurement: Keeping defense separate makes TACOS/"incremental" analysis honest
  • Stability: Brand & PDP defense should be "steady coverage," not whipsawed by discovery rules
